## Skills + standing rules updated today

None directly authored today (no `.claude/skills/*` edits), but several patterns surfaced that may warrant promotion on the next manual cron-learnings review:

1. **Cherry-pick PRs that branched from old main** — Agent DD's pattern of taking the substantive files + restoring main's version on every conflict-prone file. The "25-of-28-held-back" disposition is the right shape for stale PRs that would silently revert downstream work.
2. **Schema include validation** — Prisma rejects an `include: { foo: ... }` if the relation isn't declared on the model. The Touchpoint case (`fbde436`) is the canonical example. Check schema before referencing relations in includes.
3. **Soft-delete deletedAt filter must be on EVERY read endpoint** — list, GET /:id, search, count. Three commits today (211aea3, f600a89) fixed the same pattern in different endpoints. Standing rule worth strengthening.
