2. **Shape-preserving helper + projected-away column trap** (single instance, awaits second) — when promoting an inline reducer to a lib helper that defensively re-applies a filter, audit every callsite's Prisma SELECT for the filter-input fields. The defensive re-filter is good practice, but means callers can no longer project away the filter columns silently. Canonical example: Wave 9 `e0fa216` — the SELECT `{ id, serviceId, amountCharged, doctorId }` projected away `status`, so `pnlSumCompleted`'s `if (v.status !== 'completed') continue` short-circuited every row → `canonical.revenue = 0`.

## Pitfalls to avoid

1. **Don't dispatch agents on PRD-doc items without pre-flight grep.** The 8-May Google Doc is stale. Any agent that takes a doc item at face value will hit phantom-carry-over (4 confirmed instances).
2. **Don't run `git commit -o schema.prisma` during a parallel wave.** Sibling agents' WIP gets swept up. Wave 8's Agent B did this; tainted commit `d1db110` was reverted. Use the `.tmp-apply-schema.js` patcher pattern documented in the dispatching-parallel-agent-wave cron-learning.
3. **Don't push scripts-only commits during e2e-full.** v3.6.0's e2e-full attempt 2 went red because a mid-run `seed-drugs-on-demo.py` push triggered a deploy that pm2-restarted the backend mid-test. Root cause: `scripts/**` not in `paths-ignore`. Worth a future deploy.yml extension OR just push scripts at quiet times.
