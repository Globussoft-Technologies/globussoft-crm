### 5. Cron-learnings logged (`b18a6c9`)

Three new entries in CLAUDE.md's "Cron learnings" section:

1. **Spec rot from intentional backend hardening is the dominant e2e-full failure mode.** For any backend change that adds a validator / hardens a shape / refactors a credential / adds an SSRF guard, grep `e2e/tests/` for the field name AND endpoint path AND update every spec that touches it — not just the route's primary `*-api.spec.js`.
2. **`test.describe.configure({ mode: 'serial', timeout: 120_000 })` is the right primitive for tests sharing a backend resource under concurrent shard load.** Trade a few seconds of test wall-clock for ~2× stability.
3. **Demo-state convergence helpers must ACT every iteration, not just observe.** Under continuous background-cron writes, polling-without-acting loses; polling-and-re-acting wins.
